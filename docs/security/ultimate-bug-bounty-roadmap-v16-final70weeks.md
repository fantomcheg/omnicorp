## ⏱️ Формат обучения: Sustainable Learning Approach

> **3×90 минут в неделю** — практичный подход для предотвращения выгорания

### Session A (90 min) — Hands-on / Lab
- **60 min** — лабораторная работа (PortSwigger / PentesterLab / локальная среда)
- **20 min** — записать PoC команды (curl/Burp) + скриншоты/логи
- **10 min** — ретроспектива: что узнал, что нужно повторить

### Session B (90 min) — Теория + чтение + разбор
- **30–45 min** — теоретический материал (RFC, nginx docs, OWASP)
- **30 min** — туториал/разбор PoC
- **15–30 min** — конспект + checklist для повторного теста

### Session C (90 min) — Интеграция / автоматизация / отчет
- **45 min** — автоматизация / повторение PoC (скрипт, docker-compose)
- **30 min** — оформить краткий результат (1–2 параграфа remediation + PoC) в MD
- **15 min** — план на следующую неделю

> **Гибкость:** При нехватке времени заменяй на 2×90 или 2×45 минут — главное регулярность, не интенсивность.

## 🎯 Ключевые Принципы

1. **Consistency > Intensity** — регулярность важнее интенсивности
2. **Quality > Quantity** — 1 critical bug > 10 low severity
3. **Depth > Breadth** — мастерство в нише > поверхностное знание всего
4. **Automation > Manual** — инвестиции в tooling окупаются
5. **Community > Solo** — networking ускоряет рост

### Attack Chains Philosophy
- [ ] Учиться связывать несколько мелких багов (low-severity) в одну серьёзную атаку
- [ ] Читать ресёрчи (Orange Tsai, Assetnote, Doyensec) не просто «для галочки»
- [ ] Пробовать воспроизводить куски и писать свои модули
- [ ] Развивать mindset ресёрча

### Review & Adjust
- **Каждые 12 недель:** Review & Adjust
- **При выгорании:** Снизить интенсивность, не бросать
- **При достижении целей раньше:** Поднять планку
- **При новых возможностях:** Pivot (новая программа, research topic)

if __name__ == "__main__":

#!/bin/bash
# recon-subdomains.sh
TARGET=$1
OUTPUT_DIR="recon_${TARGET}"
mkdir -p ${OUTPUT_DIR}

echo "[*] Starting subdomain enumeration for ${TARGET}"

# Multiple sources
subfinder -d ${TARGET} -o ${OUTPUT_DIR}/subfinder.txt

# Certificate transparency
curl -s "https://crt.sh/?q=%25.${TARGET}&output=json" | jq -r '.[].name_value' | sed 's/\*\.//g' | sort -u > ${OUTPUT_DIR}/crt.txt

# Combine and deduplicate
cat ${OUTPUT_DIR}/*.txt | sort -u > ${OUTPUT_DIR}/all_subdomains.txt

echo "[+] Found $(wc -l < ${OUTPUT_DIR}/all_subdomains.txt) unique subdomains"

## Reporting Automation

### Generate Summary Report
```python
#!/usr/bin/env python3
# generate-report.py

import json
import sys
from datetime import datetime

def generate_report(nuclei_json, output_file):
    with open(nuclei_json, 'r') as f:
        findings = [json.loads(line) for line in f]
    
    # Group by severity
    by_severity = {}
    for finding in findings:
        sev = finding.get('info', {}).get('severity', 'unknown')
        by_severity.setdefault(sev, []).append(finding)
    
    # Generate markdown report
    report = f"# Security Scan Report\n\n"
    report += f"**Date:** {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}\n\n"
    report += f"**Total Findings:** {len(findings)}\n\n"
    
    for severity in ['critical', 'high', 'medium', 'low']:
        if severity in by_severity:
            report += f"## {severity.upper()} ({len(by_severity[severity])})\n\n"
            for finding in by_severity[severity]:
                report += f"### {finding['info']['name']}\n"
                report += f"- **Host:** {finding.get('host', 'N/A')}\n"
                report += f"- **Description:** {finding['info'].get('description', 'N/A')}\n"
                report += f"- **Matched:** {finding.get('matched-at', 'N/A')}\n\n"
    
    with open(output_file, 'w') as f:
        f.write(report)
    
    print(f"[+] Report generated: {output_file}")

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print(f"Usage: {sys.argv[0]} <nuclei.json> <output.md>")
        sys.exit(1)
    
    generate_report(sys.argv[1], sys.argv[2])
```
