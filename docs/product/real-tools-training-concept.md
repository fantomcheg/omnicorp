# Real Tools Training Concept

## Главный принцип
OmniCorp не подменяет реальные инструменты фейковыми.

Игра дает:
- сюжет;
- цель;
- контекст;
- проверку результата;
- короткие подсказки;
- прогресс и атмосферу.

Игрок использует реальные инструменты:
- браузер;
- DevTools;
- `curl`;
- Burp Suite;
- OWASP ZAP;
- терминал;
- реальные HTTP-запросы.

## Позиционирование
OmniCorp - это киберпанк-игра по веб-безопасности, где игрок решает сюжетные миссии настоящими инструментами.

Короткая англоязычная формулировка:

```text
Cybersecurity game where you use real web security tools.
```

## Что должно быть настоящим
- Инструменты.
- Уязвимости.
- HTTP-синтаксис.
- Команды.
- Payloads.
- Proof/flag/result.
- Remediation.

## Что может быть вымышленным
- Мир.
- Корпорации.
- Персонажи.
- Сюжет.
- Названия внутренних сервисов.
- Мотивация миссий.
- Атмосфера.

## Пример правильного подхода
Не так:

```text
hackport 80
HACK_PROFILE --target user_1002
```

А так:

```bash
curl -i 'http://localhost:3001/profile?id=1002'
nmap -sV -p 80,443 lab.local
```

Или через реальные инструменты:
- открыть браузер;
- открыть DevTools;
- перехватить запрос в Burp или ZAP;
- изменить параметр;
- повторить запрос;
- получить proof;
- отправить proof в игру.

## Игровая оболочка
Игровой интерфейс не должен быть фейковой заменой реальных действий.

Допустимо:
- OmniTerminal как подсказчик;
- OmniProxy как визуальная оболочка над реальными HTTP-запросами;
- Story UI как постановка цели и контекста;
- Mission UI как проверка proof/result.

Недопустимо:
- магические команды вместо реальных;
- вымышленные сканеры, которые скрывают суть;
- кнопка "взломать сервер";
- синтаксис, который нельзя перенести в реальную практику.

## Два режима миссий

### Training Mode
Для новичков.

Особенности:
- больше подсказок;
- запросы могут показываться внутри UI;
- игра объясняет, что искать;
- прогрессия мягкая;
- цель - понять механику уязвимости.
- проверка может быть через выбор ответа, как в текущем прототипе.

### Real Mode
Для тех, кто уже учится AppSec.

Особенности:
- игра дает цель и URL лаборатории;
- игрок сам выбирает инструмент;
- игра не диктует способ решения;
- завершение через proof или flag;
- цель - приблизить опыт к реальному web security workflow.

## Разница между Training Mode и Real Mode

Сюжет может быть в обоих режимах.

Разница не в наличии сюжета, а в уровне самостоятельности:

- `Training Mode` - теория, guided practice, варианты ответа, подсказки, remediation.
- `Real Mode` - цель, URL лаборатории, реальные инструменты, proof/flag.

## Проверять результат, а не способ
OmniCorp должна проверять не конкретный инструмент, а итог.

Пример:

```text
Цель: докажи, что портал сотрудников уязвим к IDOR.
```

Игрок может:
- изменить `id` в браузере;
- использовать DevTools;
- повторить запрос через Burp Repeater;
- проверить через OWASP ZAP;
- выполнить `curl`;
- написать маленький скрипт.

Игра проверяет:
- введен правильный proof;
- получен нужный флаг;
- выполнено условие миссии.

## Пример миссии

OmniCorp Mission UI:

```text
Цель: докажи, что портал сотрудников уязвим к IDOR.
Сервис: http://localhost:3001/profile?id=1001
Proof: найди proof из чужого профиля сотрудника.
```

Возможное решение:

```bash
curl -i 'http://localhost:3001/profile?id=1002'
```

Игра не требует именно `curl`. Она принимает proof.

## Целевой ориентир
OmniCorp должен быть ближе к:
- PortSwigger Academy;
- Hacknet;
- практическим AppSec labs;
- киберпанк-сюжету.

Сильная сторона:
- атмосфера и сюжет Hacknet;
- реальные инструменты и практическая ценность PortSwigger Academy.

Слабое место, которого нужно избегать:
- фейковые команды;
- магическая симуляция взлома;
- обучение, которое нельзя перенести в реальный workflow.
