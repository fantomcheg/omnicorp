# План первой недели OmniCorp

## Цель
Сделать браузерный прототип игры, который позже можно упаковать в desktop и Steam без переписывания ядра.

## Направление продукта
- Сначала делаем веб-приложение, но не как маркетинговый сайт.
- Строим игровой прототип с учебными миссиями.
- Сразу держим ядро независимым от платформы, чтобы потом без боли перенести его в Electron и Steam.
- Закладываем локализацию с самого начала.

## Целевая архитектура

### 1. `game-core`
- Правила игры
- Логика миссий
- Отслеживание прогресса
- Инвентарь и статусы
- Проверки уязвимостей и подсчет результата

### 2. `content`
- Сюжет
- Диалоги
- Миссии
- Локализация
- Конфиги уровней

### 3. `ui`
- HUD
- Меню
- Экраны миссий
- Экраны результатов и подсказок

### 4. `platform-web`
- Запуск в браузере
- Сохранение через `localStorage` или `IndexedDB`
- Загрузка ассетов
- Адаптация под размер экрана

### 5. `platform-desktop`
- Позже оболочка на Electron
- Доступ к файловой системе
- Локальные сохранения
- Настройки
- Упаковка под Steam

## Главный принцип
- `game-core` не должен знать, где он запущен: в браузере или на десктопе.
- Не привязываем core-логику напрямую к `window`, `document` или `localStorage`.
- Платформенные вещи идут только через адаптеры.

## Стек на первую неделю
- `Vite`
- `TypeScript`
- `PixiJS`
- Локализация `EN + RU` с первого дня
- Минимальный слой хранения состояния
- Без Electron на первой неделе
- Без тяжелого backend на первой неделе

## Результаты первой недели
1. Инициализировать структуру проекта.
2. Вывести пустую сцену в браузере.
3. Добавить один тестовый уровень.
4. Добавить движение персонажа.
5. Добавить один интерактивный объект.
6. Добавить один экран миссии.
7. Добавить локализацию `EN + RU`.
8. Добавить простой слой сохранения прогресса.

## Порядок работ на первую неделю
1. Поднять `Vite + TypeScript`.
2. Подключить `PixiJS`.
3. Вывести первую сцену.
4. Реализовать движение.
5. Добавить одну цель миссии.
6. Добавить базовый HUD с текстом.
7. Добавить ключи локализации и загрузчики.
8. Добавить абстракцию для сохранения и загрузки.

## Что пока не делать
- Не строить полноценную веб-оболочку.
- Не подключать Electron сейчас.
- Не делать большой backend сейчас.
- Не смешивать игровую логику с админскими инструментами.
- Не хардкодить тексты в коде.

## Путь к Steam
- Сначала веб-версия для быстрого тестирования.
- Потом desktop-версия через Electron.
- Один и тот же `game-core`, один и тот же контент, одна и та же UI-логика.
- Steam-релиз — это упаковка, а не переписывание.

## Практический итог первой недели
- Проект запускается в браузере.
- Персонаж может двигаться по экрану.
- Есть одна маленькая миссия.
- Контент готов к локализации.
- Кодовая база готова к будущей desktop-версии.
